# 記帳 App — GitHub Pages 版部署教學

這版把前端放 GitHub Pages、資料用 `fetch` 打 Apps Script API（**用試算表 ID 綁定你的「記帳本」**）。
好處：自己的 icon 一定生效、加到主畫面/ Dock 穩定、不再有「資料寫進去卻讀不回來」的問題。

需要的檔案（都在這個資料夾裡）：
`index.html`、`manifest.json`、`icon-180.png`、`icon-192.png`、`icon-512.png`、`記帳_API_Code.gs`

---

## Part 1 — 先部署後端 API（拿到網址）

1. 到 [script.google.com](https://script.google.com) → **新增專案**。
2. 把預設 `Code.gs` 內容全刪，貼上 **`記帳_API_Code.gs`** 的完整內容。
   - 裡面第 13 行的 `SHEET_ID` 已經填好你的「記帳本」ID，不用改。
3. 右上 **部署 → 新增部署作業** → 齒輪 ⚙️ 選「**網頁應用程式**」。
   - 執行身分：**我**
   - 誰可以存取：**任何人**
   - → **部署**。
4. 跳授權就一路允許（會要存取試算表＋雲端硬碟，因為要寫資料、存收據）。看到「未驗證」→ 進階 → 前往 → 允許。
5. 複製結尾 **`/exec`** 的網址（長得像 `https://script.google.com/macros/s/XXXX/exec`）。**這條等下要貼到前端。**

> 先測一下：把這條 `/exec` 網址直接貼到瀏覽器打開，應該看到一段 JSON（`{"items":[...],"config":{...}}`）。看到 JSON 就代表 API 正常。

---

## Part 2 — 把 `/exec` 網址填進 index.html

打開 `index.html`，找到最上面這行（約在 `<script>` 開頭）：

```js
var API_URL = "PASTE_API_URL_HERE";
```

把 `PASTE_API_URL_HERE` 換成你剛剛複製的 `/exec` 網址，例如：

```js
var API_URL = "https://script.google.com/macros/s/XXXX/exec";
```

存檔。（也可以先上傳到 GitHub 後，再到 GitHub 上點 index.html → 鉛筆編輯，直接改這行。）

---

## Part 3 — 上 GitHub Pages

1. 到 [github.com](https://github.com) 註冊／登入 → 右上 **＋ → New repository**。
   - Repository name 隨意（例如 `records`），選 **Public**，建立。
2. 進到 repo → **Add file → Upload files** → 把這個資料夾裡的
   `index.html`、`manifest.json`、`icon-180.png`、`icon-192.png`、`icon-512.png`
   一起拖進去 → 下方 **Commit changes**。
   （`記帳_API_Code.gs`、這份教學不用上傳，那是給你自己看的。）
3. repo 上方 **Settings → Pages**：
   - Source 選 **Deploy from a branch**
   - Branch 選 **main**、資料夾 **/ (root)** → **Save**。
4. 等 1–2 分鐘，Pages 會給你一個網址，像：
   `https://你的帳號.github.io/records/`
5. 打開那個網址確認能記帳、資料有出現。

---

## Part 4 — 變成 App

- **iPhone**：Safari 開上面的 Pages 網址 → 分享 → **加入主畫面**。icon 就是小豬。
  （之前那顆黑色「記」捷徑先刪掉再加。）
- **Mac**：Safari 開網址 → **檔案 → 加入 Dock**；或 Chrome → 建立捷徑（在視窗中開啟）。

---

## 注意事項

- **`API_URL` 一定要填**，沒填的話 App 只會用本機儲存（不會雲端同步）。
- **隱私**：API 設「任何人」＝知道那條 `/exec` 網址的人就能存取你的帳。當個人/家庭用沒問題，但別把網址公開貼到網路上。
- **改版**：之後我給你新的 `index.html`，重新 Upload 覆蓋、Commit 即可，Pages 會自動更新（手機重新整理；主畫面捷徑若沒更新就刪掉重加）。
- **改後端**：若我更新 `記帳_API_Code.gs`，要回 Apps Script 貼上新內容 → 部署 → 管理部署作業 → 編輯 → 新版本 → 部署。
